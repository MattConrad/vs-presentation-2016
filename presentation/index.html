Visual Studio 101: What is Visual Studio Anyway?


Visual Studio is an IDE used to develop applications in MS languages like:

  • C#
  • VB.NET
  • Visual C++


(Occasionally also used with non-Microsoft languages, too. See, for example, Python Tools for Visual Studio.)

What is an IDE Anyway?


IDE stands for Integrated Development Environment.

Most of writing an app means writing code, and a nice code editor is a big part of what an IDE provides.

But IDEs also integrate extra tools into the editing application. IDEs build in features like:

  • file browser
  • debugger
  • language documentation
  • build tools
  • awareness of standard project structures
  • code inspectors/analyzers

There are upsides and downsides to having lots of bells and whistles. Not everyone likes IDEs.

But most people who do MS coding like Visual Studio. It is a very good IDE.


(One little thing to note. Visual Studio, and Visual Studio Code, are two different IDEs.
Tonight we are talking about Visual Studio and not Visual Studio Code. Don't be confused.)

Getting Visual Studio


Happily, Visual Studio Community is now a free download from Microsoft.


(Visual Studio Community, unlike earlier free versions of Visual Studio, is a full featured package and is good enough for most use cases.)


You can download Visual Studio Community 2015 at https://www.visualstudio.com/vs/community/.


The version of Visual Studio used in the examples tonight is Visual Studio Community 2015.

Let's Do Something With Visual Studio


Our examples tonight will use the completed Contoso University application.


(You can do a tutorial where you build this application a bit at a time, at https://www.asp.net/mvc/overview/getting-started/getting-started-with-ef-using-mvc/creating-an-entity-framework-data-model-for-an-asp-net-mvc-application.)


(You might notice we're using the 2013 version of the application instead of the 2015 version. This is deliberate, don't sweat it.)


We'll start by opening up this Solution, by clicking on the .sln file, and seeing what's on the screen.

Visual Studio is quite customizable, so what you'll see will probably be somewhat different.


There's several things you're likely to find, though:


  • A Solution Explorer pane docked either on the left or the right
  • An action menu (File, Edit, View, etc) across the top of the screen
  • The Quick Launch textbox in the upper right corner.
  • A Properties pane docked under Solution Explorer
  • A big tabbed pane (often Output) docked at the bottom


(we pause here for a brief discussion of dockability and tabbability)

Solution Explorer is very important. It shows the files in your solution! Particular things to note:


  • Different projects, and then folders/files, in the tree layout
  • Show All Files
  • Search Solution Explorer
  • Properties and References folders
  • Source control status indicators
  • Getting Solution Explorer back after you accidentally close it


Learn new and exciting things by right-clicking elements in Solution Explorer!


(You can right click all sorts of things in Visual Studio. If you can interact with it, try right clicking on it and see what pops up.)



Briefly, let's talk about some other visible elements here


  • Properties pane (below Solution Explorer)
  • Server Explorer and Toolbox (left hand slider)
  • That big tabbed pane at the bottom that I will call the Output pane but which is often the Errors pane
  • The customizable toolbar(s) at the top that I never use except to turn off Browser Link


And we should really touch briefly (briefly, there's a lot here) on some of the menu columns at the top . . .

Let's open up a code file and see some new magic


(I won't be showing any GUI designers today. Sorry! They do exist, if you're doing Winforms or sometimes for web dev.)


Many, many things to see here. Let's hit some high points:

  • Right clicking on text elements does stuff!
  • Go To Line (Ctrl G) (note also line numbers in left hand gutter)
  • Go To Definition (F12)
  • Search in File (Ctrl F, or F3) (note case sensitive, regex, match word, set scope)
  • Replace in File (Ctrl H)
  • Search (or Replace) in Solution (*), showing multiple results (Ctrl Shift F, Ctrl Shift H)
  • Find All References

Multiple tabs open has a couple of interesting/useful aspects:


  • Reordering tabs by drag and drop
  • Pinning tabs
  • Close all but this/close all but pinned
  • Ctrl F4 closes active tab
  • Ctrl Tab/Ctrl Shift Tab switches from tab to tab
